## 🧱 战斗规则一：指数伤亡模型（`rule_exponent`）

### 📌 规则标识

* `key`: `rule_exponent`

### 📛 规则名称

* **指数伤亡模型**

### 📝 规则版本

* `v1.0`

### 📖 描述

> 使用攻击方与防御方的**总攻防比值**，通过指数函数 `f(x) = (D/A)^1.422 / (1 + (D/A)^1.422)` 得出伤亡比例，适用于放置类游戏中一次性结算战斗结果，具备平滑过渡、高攻低损、高防反击效果。

---

### ⚙️ 核心机制说明

#### 1. 胜负判定

* 若攻击总值 `A > D`，则攻击方胜利，防守方全灭；
* 若 `A <= D`，默认攻击失败（当前版本不计算失败方逻辑，可扩展）。

#### 2. 损失比例计算（攻方）

```ts
// totalAttack = A，totalDefense = D
l = Math.pow(D / A, 1.422)
lossRatio = l / (1 + l)
```

#### 3. 每单位兵种损失人数

```ts
loss = Math.round(unit.count × lossRatio)
```

* 每个兵种按自己原始人数和相同的损失比例计算伤亡；
* 适用于自动征兵 + 配置类战斗，无需回合模拟。

---

### 📊 特点分析

| 特性   | 表现                                    |
| ---- | ------------------------------------- |
| 平滑性  | 采用指数函数平滑过渡，不存在“刚刚打赢就0损失”的断崖式变化        |
| 公平性  | 攻强防弱 → 损失小；防强攻弱 → 损失大                 |
| 可调节性 | 指数 `1.422` 可微调以控制整体战斗强度（建议范围：1.2～1.6） |
| 性能高  | 无需逐单位模拟，适合放置类、地图大规模战斗                 |
| 可扩展性 | 可与“攻击类型占比加权防御值”逻辑组合（你目前已采用）           |

---

### ✅ 应用建议

* 默认战斗规则，适合主线副本、NPC攻打、资源掠夺等场景；
* 可配合统一攻防字段 `totalAttack`, `totalDefense` 使用；
* 推荐作为第一阶段主力规则上线；
* 后期可加入“失败方”对攻方反击计算（同理反推即可）。

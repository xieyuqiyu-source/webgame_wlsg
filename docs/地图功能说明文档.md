# 地图功能说明文档 🗺️

## 📖 功能概述

地图功能是武林三国游戏的核心模块之一，为玩家提供了探索三国世界、与其他玩家互动、挑战副本的完整体验。该模块采用Tab切换设计，包含NPC城池、玩家城池和副本三个主要功能板块。

## 🏗️ 架构设计

### 文件结构
```
src/views/map/
├── index.vue                    # 地图主页面
├── components/
│   ├── MapTabs.vue             # Tab切换组件
│   ├── NpcList.vue             # NPC城池列表
│   ├── PlayerList.vue          # 玩家城池列表
│   └── DungeonList.vue         # 副本列表
```

### 技术栈
- **Vue 3 Composition API**：现代化的组件开发方式
- **Tailwind CSS**：响应式样式框架
- **SVG图标**：矢量图标系统，确保清晰度
- **组件化设计**：模块化开发，便于维护和扩展

## 🎯 核心功能

### 1. NPC城池系统

#### 功能特点
- **单行列表展示**：优化性能，减少渲染开销
- **智能搜索**：支持城池名称和阵营搜索
- **阵营筛选**：魏、蜀、吴三国阵营分类
- **等级筛选**：按NPC等级范围筛选
- **实时统计**：显示总数量和可攻击数量

#### 界面布局
```
[搜索栏] [阵营筛选] [等级筛选]
[统计信息：总NPC数 | 可攻击数]
[NPC列表：阵营 | 名称+等级 | 资源信息 | 操作按钮]
[分页导航]
```

#### 操作功能
- **侦查**：查看NPC详细信息
- **攻击**：发起对NPC的攻击
- **点击查看**：显示NPC详细属性

### 2. 玩家城池系统

#### 功能特点
- **在线状态显示**：实时显示玩家在线/离线状态
- **卡片式布局**：美观的玩家信息展示
- **多维度筛选**：阵营、在线状态、保护状态
- **战力评估**：显示玩家军队战斗力
- **保护机制**：新手保护状态识别

#### 界面布局
```
[搜索栏] [阵营筛选] [仅在线筛选]
[统计信息：在线玩家 | 可攻击玩家]
[玩家卡片网格：头像+阵营 | 基本信息 | 城池信息 | 军队信息 | 操作按钮]
```

#### 操作功能
- **侦查**：查看玩家城池详情
- **攻击**：发起PVP战斗（需满足条件）
- **点击查看**：显示玩家详细信息

### 3. 副本系统

#### 功能特点
- **难度分级**：简单、普通、困难、地狱四个难度
- **进度追踪**：显示副本完成状态
- **奖励预览**：展示副本奖励内容
- **条件检查**：验证挑战条件和解锁状态
- **体力消耗**：显示挑战所需体力值

#### 界面布局
```
[搜索栏] [难度筛选] [仅可挑战筛选]
[统计信息：总副本 | 可挑战 | 已完成]
[副本卡片网格：难度图标 | 基本信息 | 要求条件 | 奖励预览 | 操作按钮]
```

#### 操作功能
- **挑战副本**：开始副本战斗
- **查看奖励**：预览副本奖励
- **条件检查**：显示解锁条件

## 🔄 数据流设计

### 状态管理
```javascript
// 地图相关状态
const mapStore = {
  // NPC数据
  npcList: [],
  npcFilters: { faction: 'all', level: 'all' },
  
  // 玩家数据
  playerList: [],
  playerFilters: { faction: 'all', onlineOnly: true },
  
  // 副本数据
  dungeonList: [],
  dungeonFilters: { difficulty: 'all', availableOnly: false }
}
```

### API接口设计
```javascript
// NPC相关接口
GET /api/map/npcs          // 获取NPC列表
POST /api/map/npcs/scout   // 侦查NPC
POST /api/map/npcs/attack  // 攻击NPC

// 玩家相关接口
GET /api/map/players       // 获取玩家列表
POST /api/map/players/scout // 侦查玩家
POST /api/map/players/attack // 攻击玩家

// 副本相关接口
GET /api/map/dungeons      // 获取副本列表
POST /api/map/dungeons/challenge // 挑战副本
```

## 🎮 游戏机制

### 战斗系统
- **战力计算**：基于军队数量和装备等级
- **胜负判定**：战力对比和随机因素
- **奖励分配**：根据战斗结果分配资源
- **冷却时间**：防止频繁攻击的机制

### 保护机制
- **新手保护**：新玩家的攻击免疫期
- **离线保护**：离线玩家的部分保护
- **资源保护**：防止资源完全被掠夺

### 平衡性设计
- **等级匹配**：推荐攻击相近等级的目标
- **资源限制**：单次攻击的资源获取上限
- **体力消耗**：限制玩家的攻击频率

# 统一调度器使用文档

## 概述

统一调度器是武林三国游戏的核心任务管理系统，负责管理游戏中的各种定时任务和事件调度，包括资源增长、建筑升级、军队训练、自动保存等功能。

## 核心特性

- ✅ **任务类型支持**: 立即执行、延时执行、间隔执行、定时执行
- ✅ **优先级管理**: 低、普通、高、紧急四个优先级
- ✅ **并发控制**: 支持最大并发任务数限制
- ✅ **错误处理**: 自动重试机制和错误记录
- ✅ **状态管理**: 任务暂停、恢复、取消功能
- ✅ **事件系统**: 任务生命周期事件监听
- ✅ **性能监控**: 实时统计和监控界面
- ✅ **Vue集成**: 提供组合式函数便捷使用

## 快速开始

### 1. 基础使用

```javascript
import gameScheduler, { TASK_TYPE, TASK_PRIORITY } from '@/utils/scheduler.js'

// 启动调度器
gameScheduler.start()

// 注册一个简单的定时任务
const taskId = gameScheduler.registerTask({
  name: 'resource_growth',
  type: TASK_TYPE.INTERVAL,
  interval: 5000, // 每5秒执行一次
  priority: TASK_PRIORITY.HIGH,
  handler: () => {
    console.log('资源增长中...')
    // 执行资源增长逻辑
  }
})
```

### 2. 在 Vue 组件中使用

```vue
<template>
  <div>
    <button @click="startGameTasks">启动游戏任务</button>
    <button @click="pauseAllTasks">暂停所有任务</button>
    <div>运行中的任务: {{ runningTasks }}</div>
  </div>
</template>

<script>
import { useScheduler, useGameTasks } from '@/hooks/useScheduler.js'

export default {
  setup() {
    const { 
      isRunning, 
      runningTasks, 
      pauseAllTasks, 
      resumeAllTasks 
    } = useScheduler()
    
    const { startCoreGameTasks } = useGameTasks()

    const startGameTasks = async () => {
      await startCoreGameTasks()
    }

    return {
      isRunning,
      runningTasks,
      startGameTasks,
      pauseAllTasks
    }
  }
}
</script>
```

### 3. 使用调度器管理器

```javascript
import { schedulerManager, SchedulerExamples } from '@/utils/schedulerManager.js'

// 初始化调度器管理器
await schedulerManager.initialize()

// 注册游戏任务
schedulerManager.registerGameTask('RESOURCE_GROWTH', async () => {
  // 资源增长逻辑
  const growth = calculateResourceGrowth()
  store.dispatch('resources/addResources', growth)
})

// 暂停特定游戏任务
schedulerManager.pauseGameTask('RESOURCE_GROWTH')

// 恢复特定游戏任务
schedulerManager.resumeGameTask('RESOURCE_GROWTH')
```

## 任务类型详解

### 1. 立即执行任务 (IMMEDIATE)
```javascript
gameScheduler.registerTask({
  name: 'immediate_task',
  type: TASK_TYPE.IMMEDIATE,
  handler: () => {
    console.log('立即执行')
  }
})
```

### 2. 延时执行任务 (TIMEOUT)
```javascript
gameScheduler.registerTask({
  name: 'delayed_task',
  type: TASK_TYPE.TIMEOUT,
  delay: 3000, // 3秒后执行
  handler: () => {
    console.log('延时执行')
  }
})
```

### 3. 间隔执行任务 (INTERVAL)
```javascript
gameScheduler.registerTask({
  name: 'interval_task',
  type: TASK_TYPE.INTERVAL,
  interval: 5000, // 每5秒执行一次
  handler: () => {
    console.log('间隔执行')
  }
})
```

### 4. 定时执行任务 (CRON)
```javascript
gameScheduler.registerTask({
  name: 'cron_task',
  type: TASK_TYPE.CRON,
  handler: () => {
    console.log('定时执行')
  }
})
```

## 错误处理和重试

调度器支持自动重试机制：

```javascript
gameScheduler.registerTask({
  name: 'risky_task',
  type: TASK_TYPE.INTERVAL,
  interval: 10000,
  maxRetries: 3, // 最大重试3次
  handler: async () => {
    // 可能失败的操作
    const result = await riskyOperation()
    return result
  }
})
```

## 事件监听

## 最佳实践

### 1. 任务命名规范
- 使用描述性的任务名称
- 采用下划线分隔的命名方式
- 包含任务类型信息

```javascript
// 好的命名
'resource_growth_calculation'
'building_upgrade_check'
'army_training_completion'

// 不好的命名
'task1'
'job'
'timer'
```

### 2. 优先级设置
- **CRITICAL**: 关键系统任务（自动保存、错误处理）
- **HIGH**: 核心游戏逻辑（资源增长、战斗计算）
- **NORMAL**: 一般功能（状态检查、UI更新）
- **LOW**: 非关键任务（统计收集、日志记录）

### 3. 错误处理
```javascript
gameScheduler.registerTask({
  name: 'safe_task',
  handler: async () => {
    try {
      await someOperation()
    } catch (error) {
      console.error('任务执行失败:', error)
      // 记录错误但不抛出，避免影响调度器
      return { success: false, error: error.message }
    }
  }
})
```

### 4. 资源清理
```javascript
// 组件卸载时清理任务
onUnmounted(() => {
  gameScheduler.cancelTask(taskId)
})
```

## 游戏集成示例

### 完整的游戏初始化流程
```javascript
import { useScheduler, useGameTasks } from '@/hooks/useScheduler.js'
import { useStore } from 'vuex'

export default {
  async setup() {
    const store = useStore()
    const { initialize } = useScheduler()
    const { startCoreGameTasks, startOnlineTasks } = useGameTasks()

    // 游戏启动时的初始化流程
    const initializeGame = async () => {
      try {
        // 1. 初始化调度器
        await initialize()

        // 2. 启动核心游戏任务
        await startCoreGameTasks()

        // 3. 如果在线，启动在线任务
        if (store.getters['game/isOnline']) {
          await startOnlineTasks()
        }

        console.log('游戏初始化完成')
      } catch (error) {
        console.error('游戏初始化失败:', error)
      }
    }

    // 组件挂载时初始化
    onMounted(initializeGame)

    return {
      initializeGame
    }
  }
}
```

## 注意事项

1. **内存管理**: 及时清理不需要的任务，避免内存泄漏
2. **并发控制**: 合理设置最大并发任务数，避免性能问题
3. **错误处理**: 任务中的错误应该被妥善处理，不应影响调度器运行
4. **优先级平衡**: 避免过多高优先级任务影响系统性能
5. **任务粒度**: 任务不宜过重，避免阻塞其他任务执行

## 故障排除

### 常见问题

1. **任务不执行**
   - 检查调度器是否已启动
   - 确认任务配置正确
   - 查看任务状态是否为运行中

2. **性能问题**
   - 检查并发任务数量
   - 优化任务执行时间
   - 调整任务优先级

3. **内存泄漏**
   - 及时清理已完成的任务
   - 检查事件监听器是否正确移除
   - 避免在任务中创建循环引用

通过统一调度器，武林三国游戏可以更好地管理各种定时任务，提供稳定可靠的游戏体验。

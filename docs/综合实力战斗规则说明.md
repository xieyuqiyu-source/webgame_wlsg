# 综合实力战斗规则说明

## 概述

新的综合实力战斗规则（`COMPREHENSIVE`）是对原有战斗系统的重大优化，实现了更加真实和策略性的战斗计算。该规则分别计算步兵和骑兵的攻防能力，根据实力差距来决定损耗比例，体现了"强者损失少，弱者损失大"的战斗原则。

## 核心特性

### 1. 分兵种攻防计算

- **步兵攻击力**：所有非骑兵单位的攻击力总和
- **骑兵攻击力**：所有骑兵单位的攻击力总和
- **步兵防御力**：所有单位的步防（infantryDefense）总和
- **骑兵防御力**：所有单位的骑防（cavalryDefense）总和

### 2. 实力对比分析

系统会计算三个关键比例：
- **步兵攻防比**：进攻方步兵攻击 ÷ 防守方步防
- **骑兵攻防比**：进攻方骑兵攻击 ÷ 防守方骑防
- **综合实力比**：总攻击力 ÷ 总防御力

### 3. 智能损耗计算

根据综合实力比例，系统会智能计算损耗：

#### 实力均衡（0.8-1.25）
- 双方均等损失15%
- 体现势均力敌的激烈战斗

#### 实力差距较大
- **强势方**：最少5%损失，最多20%损失
- **弱势方**：最少15%损失，最多60%损失
- 最大优势倍数限制为4倍

### 4. 单位差异化损耗

不同单位的损耗会根据其综合实力进行调整：
- **强力单位**：攻击+步防+骑防数值高的单位损耗更少
- **弱势单位**：综合数值低的单位损耗更多
- 通过幂函数（0.3次方）进行平滑调整

## 使用示例

### 基本用法

```javascript
import { BATTLE_RULES, BATTLE_RULE_IDS } from '../config/battleRulesConfig.js'

// 获取综合实力战斗规则
const rule = BATTLE_RULES[BATTLE_RULE_IDS.COMPREHENSIVE]

// 计算战斗结果
const result = rule.calculateDamage(attackerArmy, defenderArmy)

console.log('战斗结果:', result.battleResult) // 'victory', 'defeat', 'draw'
console.log('进攻方损耗:', result.attackerLossRatio)
console.log('防守方损耗:', result.defenderLossRatio)
console.log('详细损耗:', result.attackerLosses, result.defenderLosses)
```

## 战略意义

### 1. 兵种搭配重要性
- 纯步兵军队容易被骑兵克制
- 混合编队能够应对不同威胁
- 需要根据敌方兵种构成调整自己的防御配置

### 2. 质量vs数量
- 高质量精兵在战斗中损耗更少
- 大量低质量兵种可能损失惨重
- 平衡投入成本和战斗效果

### 3. 攻防平衡
- 单纯的高攻击不一定能获胜
- 防御力同样重要，影响损耗比例
- 需要综合考虑攻防配置

## 与其他战斗规则的对比

| 特性 | 基础规则 | 指数规则 | 综合实力规则 |
|------|----------|----------|-------------|
| 兵种区分 | 简单 | 简单 | 详细分类 |
| 损耗计算 | 线性 | 指数 | 智能分级 |
| 单位差异 | 无 | 无 | 有 |
| 策略深度 | 低 | 中 | 高 |
| 计算复杂度 | 低 | 中 | 高 |

## 配置参数

可以通过修改以下参数来调整战斗规则：

```javascript
// 实力均衡范围
const BALANCE_RANGE = { min: 0.8, max: 1.25 }

// 均衡时损耗
const BALANCE_LOSS = 0.15

// 最大优势倍数
const MAX_ADVANTAGE = 4

// 损耗范围
const LOSS_RANGE = {
  winner: { min: 0.05, max: 0.2 },
  loser: { min: 0.15, max: 0.6 }
}

// 单位实力修正指数
const POWER_MODIFIER_EXPONENT = 0.3
```

## 注意事项

1. **数据完整性**：确保所有单位都有完整的攻防数据
2. **兵种类型**：正确设置 `unitType` 字段（'infantry' 或 'cavalry'）
3. **数值平衡**：注意各兵种数值的平衡性，避免某个兵种过于强势
4. **性能考虑**：复杂计算可能影响性能，大规模战斗时需要优化

# 征兵功能说明文档

## 📋 功能概述

征兵功能是武林三国游戏的核心军事系统，允许玩家消耗资源训练各种兵种来组建军队。该功能包含完整的资源管理、训练队列、时间控制和数据持久化机制。

## 🏗️ 技术架构

### 核心组件结构
```
src/
├── store/modules/gameStore.js          # 游戏状态管理（征兵核心逻辑）
├── views/military/components/
│   ├── UnitDetailTabs.vue             # 兵种详情标签页
│   └── RecruitmentModal.vue           # 征兵弹窗组件
├── config/
│   └── factionConfig.js               # 兵种配置数据

```

## 🔧 核心实现

### 1. 数据结构设计

#### gameStore 中的征兵相关数据：
```javascript
// 军队数据 - 存储各兵种的当前数量
army: {
  'wei_infantry': 10,    // 魏国步兵数量
  'shu_cavalry': 5,      // 蜀国骑兵数量
  // ... 其他兵种
}

// 征兵队列 - 正在训练的兵种列表
recruitmentQueue: [
  {
    id: 'unique_id',
    unitId: 'wei_infantry',
    count: 5,
    startTime: 1640995200000,
    endTime: 1640995500000,
    timerId: null
  }
]

// 征兵配置 - 训练时间相关设置
recruitmentConfig: {
  baseTrainTime: 300000,    // 基础训练时间（5分钟）
  speedMultiplier: 1        // 训练速度倍数（用于加速建筑）
}
```

### 2. 核心方法说明

#### 征兵相关 Actions：

**`recruitUnits(unitId, count)`**
- 功能：开始征兵流程
- 参数：兵种ID，征兵数量
- 逻辑：检查资源 → 扣除资源 → 加入队列 → 设置定时器

**`completeRecruitment(queueItem)`**
- 功能：完成征兵，将兵种加入军队
- 参数：队列项目
- 逻辑：增加军队数量 → 移除队列项 → 保存数据

**`canRecruit(unitId, count)`**
- 功能：检查是否可以征兵
- 返回：布尔值
- 检查：资源是否充足

**`getMaxRecruitableCount(unitId)`**
- 功能：计算最大可征募数量
- 返回：数字
- 逻辑：根据当前资源计算最大可训练数量

**`restoreRecruitmentTimers()`**
- 功能：游戏加载时恢复征兵定时器
- 逻辑：遍历队列 → 计算剩余时间 → 设置定时器或直接完成

#### 征兵相关 Getters：

**`totalArmyCount`**
- 功能：计算军队总数
- 返回：所有兵种数量之和

**`isRecruiting`**
- 功能：检查是否正在征兵
- 返回：布尔值

**`getActualTrainTime(unitId)`**
- 功能：计算实际训练时间
- 公式：基础时间 / 速度倍数

## ⚙️ 配置说明

### 训练时间配置

**基础训练时间：** 5分钟（300000毫秒）
- 位置：`gameStore.recruitmentConfig.baseTrainTime`
- 修改：直接修改该值即可调整所有兵种的基础训练时间

**训练速度倍数：** 1.0（默认）
- 位置：`gameStore.recruitmentConfig.speedMultiplier`
- 用途：用于实现加速建筑功能
- 示例：设置为2.0时，训练时间减半

### 兵种资源消耗配置

兵种的资源消耗在 `factionConfig.js` 中定义：
```javascript
cost: {
  wood: 100,     // 木材消耗
  clay: 80,      // 泥土消耗
  iron: 50,      // 铁消耗
  food: 30       // 食物消耗
}
```

## 🔄 数据持久化

### 保存机制
- 自动保存：每次征兵操作后自动保存
- 保存内容：军队数据、征兵队列、征兵配置
- 存储位置：localStorage

### 加载机制
- 游戏启动时自动加载数据
- 恢复征兵定时器
- 处理离线期间完成的训练

## 🐛 常见问题与解决方案

### 问题1：征兵定时器丢失
**原因：** 页面刷新或游戏重启
**解决：** `restoreRecruitmentTimers()` 方法会在游戏加载时自动恢复

### 问题2：资源不足无法征兵
**原因：** 当前资源少于兵种所需资源
**解决：** 系统会自动计算最大可征募数量并限制用户输入

### 问题3：训练时间不准确
**原因：** 系统时间变化或时区问题
**解决：** 使用 `Date.now()` 确保时间戳准确性

## 📝 维护注意事项

1. **修改训练时间：** 只需修改 `baseTrainTime` 配置
2. **添加新资源类型：** 需要同时更新兵种配置和资源检查逻辑
3. **性能优化：** 征兵队列过长时考虑分页或限制最大队列长度
4. **数据迁移：** 更新数据结构时需要考虑旧数据的兼容性

# 武林三国 - 战斗功能说明文档

## 概述

武林三国的战斗系统是一个基于回合制的策略战斗模块，支持魏蜀吴三大阵营之间的军事对抗。系统采用标准化的数据结构，支持多种战斗规则和复杂的兵种配置。

## 核心数据结构

### ArmyGroup（军团组合）

军团是战斗系统的基础单位，包含以下属性：

```javascript
{
  faction: 'wei' | 'shu' | 'wu',  // 阵营类型
  units: UnitInstance[]           // 兵种实例数组
}
```

### UnitInstance（兵种实例）

每个兵种实例包含完整的战斗属性：

```javascript
{
  id: string,              // 兵种唯一标识
  name: string,            // 兵种名称
  unitType: string,        // 兵种类型：infantry/cavalry/siege/special
  attack: number,          // 单兵攻击力
  infantryDefense: number, // 单兵步兵防御
  cavalryDefense: number,  // 单兵骑兵防御
  speed: number,           // 单兵速度
  carryCapacity: number,   // 单兵运载能力
  count: number,           // 当前出征数量
  faction: string,         // 所属阵营
  icon: string             // 展示图标（可选）
}
```

## 兵种分类系统

### 步兵营（Infantry）
- **特点**：防御力强，适合守城和正面对抗
- **代表兵种**：虎卫、青州兵、影卫等

### 骑兵营（Cavalry）
- **特点**：速度快，机动性强，适合突袭
- **代表兵种**：铁骑、神风、霸王骑等

### 攻城武器（Siege）
- **特点**：攻击力极高，专门用于攻城
- **代表兵种**：冲撞车、巨石车、霹天雷等

### 特殊兵种（Special）
- **运输单位**：木牛流马等，提供运载能力
- **贵族单位**：汉室宗亲等，提供特殊加成
- **商贸单位**：各国商人，用于经济活动
- **侦察单位**：密探等，提供情报支持

## 战斗规则系统

### 基础战斗规则（BASIC）
- **描述**：简单的攻防计算，适合快速战斗
- **计算方式**：总攻击力 vs 总防御力
- **适用场景**：日常战斗、快速测试


### 掠夺战斗规则（PLUNDER）
- **描述**：专门用于掠夺战斗，考虑运载能力
- **特殊机制**：
  - 胜利后可根据运输单位的运载能力总和，确定可掠夺的资源数量
  - 掠夺数量受运载能力限制
  - 资源按比例分配：木材、土壤、铁矿、粮食
- **适用场景**：资源掠夺、经济战争

### 指数伤亡规则（EXPONENT）
- **描述**：使用指数公式计算损耗比例
- **计算公式**：损耗比例 = (防御力/攻击力)^1.422
- **特点**：更真实的战损模拟
- **适用场景**：模拟真实战争、高难度挑战

## 阵营特色系统

### 魏国（WEI）
- **特色**："以智取胜，兵贵神速"
- **加成**：取阵营配置中的军事加成
- **特殊兵种**：土族、魏国商人
- **战术风格**：快速突击，智谋制胜

### 蜀国（SHU）
- **特色**："仁义之师，坚韧不拔"
- **加成**：取阵营配置中的军事加成
- **特殊兵种**：贪狼营、麒麟卫、青龙军、木牛流马、汉室宗亲
- **战术风格**：稳扎稳打，持久作战

### 吴国（WU）
- **特色**："水战精通，灵活机动"
- **加成**：取阵营配置中的军事加成
- **特殊兵种**：影卫、修罗、风水师、太平士
- **战术风格**：灵活机动，出其不意

## 战斗计算流程

### 1. 数据验证
```javascript
const validation = validateBattleData(attackerArmy, defenderArmy)
if (!validation.valid) {
  throw new Error(validation.errors.join(', '))
}
```

### 2. 行军时间计算
```javascript
const marchTime = BATTLE_TIME_CONFIG.calculateMarchTime(distance, attackerArmy)
```

### 3. 战斗模拟（测试战斗时需要）
```javascript
const battleResult = simulateBattle(attackerArmy, defenderArmy, ruleId)
```

### 4. 结果处理
- 计算双方伤亡
- 评估战斗效率
- 处理掠夺资源（如适用）
- 生成战斗报告

## API 使用指南

### 创建军团

```javascript
import { createSingleUnitArmy, createMultiUnitArmy } from '@/utils/battleUtils.js'

// 创建单兵种军团
const army = createSingleUnitArmy('wei', 'huWei', 100)

// 创建多兵种军团
const multiArmy = createMultiUnitArmy('shu', [
  { unitId: 'tanLangYing', count: 50 },
  { unitId: 'qiLinWei', count: 30 }
])
```

### 模拟战斗

```javascript
import { simulateBattle, BATTLE_RULE_IDS } from '@/modules/battle/battleCore.js'

// 基础战斗

// 掠夺战斗

```

### 战斗结果解析

```javascript
const {
  result,           // 战斗结果：victory/defeat/draw
  attackerDamage,   // 攻击方造成伤害
  defenderDamage,   // 防御方造成伤害
  casualties,       // 详细伤亡信息
  statistics        // 战斗统计数据

```

## 常见问题

### Q: 如何处理空军团？
A: 系统会自动验证军团数据，空军团将返回验证失败。

### Q: 掠夺资源如何分配？
A: 按运载能力限制，然后根据敌方现存资源占比进行分配。如果我运载能力总和大于敌方所有资源，就全部掠夺完毕。

### Q: 如何添加新的阵营？
A: 需要在 `factionConfig.js` 中添加阵营配置，并更新 `FACTION_TYPES` 常量。

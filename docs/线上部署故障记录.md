# 线上部署故障记录

## 2026-05-18 玩家城池 502

### 现象

- `玩家城池` 页面请求失败
- 线上接口：
  - `GET /api/health`
  - `GET /api/players`
- 均返回 `502 Bad Gateway`

### 直接原因

在 `feat: add player scouting` 阶段，`server/save-api.mjs` 曾直接引用前端文件：

```js
import { getUnitById } from '../src/config/factionConfig.js'
```

线上部署时，后端实际只会同步：

```text
/srv/wlsg-dev/server
```

不会同步：

```text
/srv/wlsg-dev/src
```

因此线上 `save-api` 进程启动失败，Nginx 代理到 `127.0.0.1:18790` 时返回 `502`。

### 已做修复

提交：

```text
d0e1205 fix: keep save api self contained
```

修复方式：

- 服务端只返回玩家侦查所需的 `兵种 id + 数量`
- 前端收到结果后，再用本地兵种配置补齐名称和属性
- `server/` 不再依赖 `src/`

### 为什么 GitHub Actions 显示失败，但线上功能后来又恢复了

自动部署脚本的顺序是：

1. 同步前端 `dist/`
2. 同步后端 `server/`
3. 重启 `wlsg-save-api`
4. 执行 `nginx -t`
5. reload Nginx

本次修复部署时，前 3 步已经成功，所以：

- 修复后的 `save-api` 已同步
- `wlsg-save-api` 已重新启动
- 线上接口随后恢复可用

但第 4 步失败，导致 GitHub Actions 最终仍显示 `failure`。

### 当前仍存在的部署阻塞

GitHub Actions 日志显示：

```text
invalid number of arguments in "proxy_set_header" directive
in /etc/nginx/conf.d/mobilewlsg.ccoos.cn.conf:10
```

这说明服务器上的正式配置文件：

```text
/etc/nginx/conf.d/mobilewlsg.ccoos.cn.conf
```

第 10 行存在语法错误。

仓库中的模板文件：

```text
deploy/nginx/mobilewlsg.ccoos.cn.conf.example
```

内容是正常的，所以问题在**服务器上的实际配置**，不在仓库模板。

### 影响

- 线上当前接口可以恢复正常
- 但 `dev-wlsg` 自动部署会继续显示失败
- 桌面站和移动站部署都会被同一个 Nginx 配置错误拦住
- 以后合并其他分支或继续推送时，如果看到部署失败，优先检查这个问题

### 服务器侧排查命令

查看出错配置：

```bash
sudo nl -ba /etc/nginx/conf.d/mobilewlsg.ccoos.cn.conf | sed -n '1,20p'
```

检查 Nginx：

```bash
sudo nginx -t
```

查看 API 服务：

```bash
sudo systemctl status wlsg-save-api.service --no-pager
sudo journalctl -u wlsg-save-api.service -n 80 --no-pager
```

### 正常的代理头配置应类似

```nginx
proxy_set_header Host $host;
proxy_set_header X-Real-IP $remote_addr;
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
proxy_set_header X-Forwarded-Proto $scheme;
```

### 后续建议

1. 修正服务器上的 `mobilewlsg.ccoos.cn.conf`
2. 重新执行：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

3. 之后再观察 `dev-wlsg` 自动部署是否恢复为绿色
4. 后续所有后端代码都应保持 `server/` 自包含，除非部署脚本同步范围也同步调整
